### 재미있는 이야기 기반 실습 문제 (5문제)
아래는 요청하신 대로, 이야기 기반의 실습문제를 구체적으로 작성한 것입니다. AJAX와 Spring Boot을 활용한 문제들입니다.

### 1. **쇼핑카트 결제 모험**

**상황**: 당신은 전자상거래 웹사이트의 개발자입니다. 고객이 상품을 장바구니에 담고 결제할 때, AJAX를 사용하여 상품 가격을 동적으로 계산하고 싶습니다. 사용자가 상품을 선택할 때마다 실시간으로 총 가격을 계산하여 화면에 표시합니다.

**실습문제**:

* 사용자가 상품을 장바구니에 추가하면, 그때마다 상품의 가격과 수량을 AJAX로 서버로 전송하고, 서버에서 계산된 총 가격을 클라이언트에 반환합니다.
* Spring Boot에서는 장바구니에 담긴 상품들의 총합을 계산하여 응답합니다.

---

### 2. **학생 성적 시스템**

**상황**: 학교의 온라인 성적 관리 시스템을 개발 중입니다. 교사가 학생의 성적을 입력하고, 이 데이터를 서버로 보내면 시스템은 해당 학생이 합격인지 불합격인지를 자동으로 계산합니다. 이 계산 결과는 페이지를 새로 고침하지 않고 바로 확인할 수 있어야 합니다.

**실습문제**:

* 교사가 성적을 입력하고 '성적 확인' 버튼을 클릭하면, AJAX를 통해 서버로 데이터를 전송하고, 서버에서 점수를 기준으로 합격 여부를 반환합니다.
* Spring Boot에서 점수 계산을 처리하고, 합격 여부를 JSON 형식으로 반환합니다.

---

### 3. **음악 플레이리스트 관리**

**상황**: 당신은 음악 스트리밍 서비스를 개발하고 있습니다. 사용자가 즐겨 듣는 곡들을 자신의 플레이리스트에 추가할 수 있습니다. 사용자는 페이지를 새로 고침하지 않고, 자신만의 플레이리스트를 실시간으로 관리할 수 있어야 합니다.

**실습문제**:

* 사용자가 '곡 추가' 버튼을 클릭하면, AJAX를 통해 선택한 곡이 플레이리스트에 추가되고, 서버에서 플레이리스트를 반환합니다.
* Spring Boot에서 플레이리스트 정보를 관리하고, 사용자가 선택한 곡을 목록에 추가한 후, 업데이트된 목록을 클라이언트에 반환합니다.

---

### 4. **상품 재고 관리 시스템**

**상황**: 당신은 온라인 쇼핑몰의 재고 관리 시스템을 개발 중입니다. 관리자가 새로운 상품을 추가하거나 기존 상품의 재고를 수정할 수 있어야 하며, 페이지 새로 고침 없이 실시간으로 재고 변동을 반영해야 합니다.

**실습문제**:

* 관리자가 상품의 이름, 가격, 수량을 입력하고 '추가' 버튼을 클릭하면, AJAX로 서버에 상품 데이터를 전송하고, 서버에서 성공적으로 상품이 추가되었다는 응답을 반환합니다.
* Spring Boot에서 상품 정보를 저장하고, 성공적인 추가 여부를 클라이언트에 알려줍니다.
