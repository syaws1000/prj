
## ✅ [객관식 15문제]

1. HTML5에서 웹 페이지 구조를 명확하게 표현하기 위해 도입된 태그는 무엇인가?  
   #① 시맨틱 태그  
   ② 스타일 태그  
   ③ 링크 태그  
   ④ 디버그 태그

2. 다음 중 시맨틱 태그에 해당하지 않는 것은?  
   ① `<section>`  
   ② `<article>`  
   #③ `<div>`  
   ④ `<aside>`

3. `<nav>` 태그는 보통 어떤 용도로 사용되는가?  
   ① 이미지 출력  
   #② 내비게이션 링크 모음  
   ③ 본문 글 정리  
   ④ 페이지 푸터 구성

4. 다음 중 `<footer>`에 주로 포함되지 않는 정보는?  
   ① 저작권 정보  
   ② 연락처  
   #③ 페이지 제목  
   ④ 작성자 이름

5. CSS에서 `display: flex;`는 어떤 레이아웃을 구성할 때 사용하는가?  
   ① 요소를 겹치게 할 때  
   ② 요소를 세로로 나열할 때  
   #③ 요소를 가로로 정렬할 때  메뉴나 수평으로 나열할 때 사용되는 속성
   ④ 요소를 투명하게 만들 때

6. 다음 중 `list-style-type: none;` 속성이 적용된 결과는?  
   ① 글자 색이 회색으로 바뀐다  
   #② 리스트 앞의 마커가 제거된다   ul/li 태그로 마커를 제거할 때, 주로 사용된다.
   ③ 리스트 항목 간 간격이 줄어든다  
   ④ 리스트 항목에 밑줄이 생긴다

7. 다음 중 링크에 마우스를 올렸을 때 효과를 주는 CSS 선택자는?  
   ① a:active  
   ② a:visited  
   #③ a:hover  
   ④ a:focus

8. Oracle SQL에서 `_` (언더바)의 의미는 무엇인가?  
   ① 공백 문자  
   ② 와일드카드로 모든 문자  
   #③ 한 글자만을 의미하는 와일드카드  글자수와 위치를 정확하게 지켜야 할 때, 사용되는 형식..
   ④ 숫자만 허용

9. 다음 SQL에서 결과에 포함되는 ENAME은 어떤 조건을 만족하는가?  
   `SELECT * FROM EMP WHERE ENAME LIKE 'S%';`  
   ① ENAME이 S로 끝나는 사원  '%S'
   ② ENAME에 S가 포함된 사원  '%S%'
   #③ ENAME이 S로 시작하는 사원   'S%' 
   ④ ENAME이 정확히 'S'인 사원   'S'

10. 다음 SQL 중 JOB의 길이가 5자리이고 세 번째 문자가 E인 데이터를 찾는 것은?  
   ① `LIKE 'E____'`  
   #② `LIKE '__E__'`  
   ③ `LIKE '_E___'`  
   ④ `LIKE '%E___'` 뒤에서 4번째자리가 E인 데이터
   			

11. `SELECT * FROM EMP ORDER BY SAL DESC;` 쿼리의 기능은?  
   ① 사원명을 오름차순으로 정렬  
   #② 급여를 내림차순으로 정렬  
   ③ 급여를 오름차순으로 정렬  
   ④ 부서번호를 기준으로 정렬

12. SQL에서 기본 정렬 옵션은 무엇인가?  
   #① ASC (오름차순)  ORDER BY 컬럼명 [ASC]  생략해도 오름차순으로 기본적으로 처리된다.
   ② DESC (내림차순)  
   ③ RANDOM  
   ④ GROUP

13. 다음 중 `ORDER BY JOB DESC, SAL` 쿼리는 어떤 정렬 순서를 의미하는가?  
   ① JOB 오름차순, SAL 오름차순  
   ② JOB 내림차순, SAL 내림차순  
   #③ JOB 내림차순, SAL 오름차순  
   ④ JOB 오름차순, SAL 내림차순

14. SQL에서 `LIKE '%R_'` 조건은 어떤 데이터를 찾는가?  
   ① R로 시작하는 데이터  
   ② R로 끝나는 데이터  
   #③ R이 마지막에서 두 번째에 위치한 데이터  
   ④ R이 포함된 모든 데이터

15. HTML에서 시맨틱 태그를 사용할 때 가장 큰 이점은 무엇인가?  
   ① 브라우저 호환성이 높아진다  
   ② 디자인 요소를 개선한다  
   #③ 문서의 의미를 명확하게 하고 검색 엔진 최적화에 도움이 된다  
   ④ 사용자 입력 처리가 쉬워진다
