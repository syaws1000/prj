# 📘 CSS 테이블 및 폼 요소 스타일링  
> **"너무 딱딱한 테이블과 폼은 이제 그만!"**  
> _— 웹사이트에 감성을 불어넣는 스타일링의 마법!_

---

### ✅ 1. 주의 집중 끌기 및 학습 연결성 제공 (Gagné 1단계 + ARCS - Attention, Relevance)

> **🎬 오프닝 퀴즈 & 짧은 영상**
- "다음 중 보기에 가장 심심한 웹페이지는?"  
- 실제 `table`과 `form`을 꾸미지 않은 화면 vs 꾸민 화면 비교 영상  
- 학습자의 **호기심과 몰입 유도**: _“우와, 저게 CSS만으로 가능한 거야?”_

> **💡 관련성 제공**
- “실제 쇼핑몰, 예약 시스템, 설문조사 폼 등은 CSS 스타일링 없이는 고객 경험이 
떨어질 수밖에 없습니다.”

---

### 🎯 2. 학습 목표 (Gagné 2단계)

```markdown
✔ CSS를 활용해 표(table)를 꾸밀 수 있다.  
✔ 폼(form) 요소의 시각적 효과를 조정할 수 있다.  
✔ 다양한 폼 요소에 대한 스타일 속성을 활용해 실무에 적용할 수 있다.
```

---

### 🔄 3. 선수 학습 상기 및 실생활 연결하기 (Gagné 3단계 + ARCS - Relevance)

> **💬 퀴즈로 회상**
- "HTML로 `<table>`, `<form>`, `<input>` 써본 적 있나요?"  
- 이전 학습 연결 질문: “HTML에서 `<table>`과 `<form>`의 기본 구조를 떠올려 봅시다!”

> **🌍 실생활 사례 연결**
- "항공권 예약 시 어떤 입력폼이 눈에 잘 띄었나요?"
- "예쁘게 꾸며진 표는 정보를 더 잘 읽게 도와줍니다."

---

### 🧠 4. 주요 개념/핵심 키워드를 최고의 암기법에 의한 제시 (Gagné 4단계 + ARCS - Confidence)

> **🧲 마법의 키워드 기억법**  
| 개념 | 기억 코드 | 예시 |
|------|-----------|------|
| `border-collapse` | "붙여붙여 테이블!" | 테두리 간격 없애기 |
| `text-align`, `vertical-align` | "가로세로 정렬파워!" | 텍스트 위치 맞추기 |
| `:focus` | "내가 누르면 집중!" | 포커스 시 스타일 변경 |
| `input[type=submit]` | "나는 버튼이다!" | 제출 버튼 꾸미기 |
| `placeholder` | "여긴 힌트박스!" | 안내 문구 스타일링 |

> **🎶 리듬기억법 예시**  
> 🎵 “가로정렬 `text-align`, 세로정렬 `vertical-align`,  
> 버튼엔 색과 그림자, 포커스 줄 땐 `:focus` 잊지마~”

---

### 🔍 5. 학습 목표별 탐구형 학습 (Gagné 5단계 + ARCS - Confidence)

> #### 📘 탐구 주제1: 테이블을 예쁘게 만들어보자!
- 미션: 기본 테이블을 `border`, `padding`, `hover 효과`로 꾸미기  
- 확장: 짝수행 배경색 변경 `tr:nth-child(even)` 활용

> #### 📘 탐구 주제2: 폼요소에 감성을 담자!
- 미션: `input`, `select`, `textarea`, `button`에 각각 스타일 적용  
- 도전: `:focus` 효과로 입력 시 애니메이션 효과 넣기  

---

### 🧑‍🤝‍🧑 6. 실습 활동 및 협력 학습 유도 (Gagné 6단계 + ARCS - Satisfaction)

> **👨‍🔧 실무형 팀 미션**
- 팀별로 "가상 쇼핑몰 주문 양식" 제작  
- 조건: 테이블로 상품 리스트 구성 + 입력 폼 꾸미기

> **💬 웃긴 요소 추가 예시**
- "입력칸 클릭 시 `background-color`를 무지개로 바꾸면 어떨까?"  
- "버튼 누르면 살짝 튀어나오게 `box-shadow`를 줘보자!"

### 💡 7. 단계별 피드백 제공과 자기 평가 기회 (Gagné 7단계)

> **✔️ 체크리스트 기반 자가 점검**
- [ ] 테이블 행마다 색이 잘 구분되는가?  
- [ ] 폼 요소의 입력 영역이 보기 편한가?  
- [ ] 포커스 시 변화가 잘 보이는가?

> **🧑‍🏫 튜터 피드백**
- 각 실습 후 팀별 발표 → 간단한 피드백 (재미 포인트도 강조)

---

### 🧪 8. 수행 평가 및 피드백 제공 (Gagné 8단계)

> **📌 수행 과제**
- 실제 구직 사이트의 지원서 폼을 참고해 나만의 입사지원서 폼 만들기  
- 조건: 스타일링 강조, 사용자의 가독성과 편의성 고려

> **📬 피드백 양식 제공**
- 디자인, 접근성, 창의성, 코드 정리도 포함한 루브릭 제공
