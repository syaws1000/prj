## 🌟 실무 기반 웹프로젝트 학습 교안  
### 《주제: 요구사항정의서부터 자바단 구현까지 웹프로젝트 실습》

---

### 1. 🧠 주의 집중 끌기 및 학습 연결성 제공  
- **ARCS-A(Attention)**  
🕵️‍♂️ “개발자 김코딩이 사라졌다?! 그의 요구사항 정의서를 복구하라!”  
▶ 도입 애니메이션/짧은 개그극: 실무 중간에 요구사항 정의서가 날아간 상황.  
▶ 학습자에게 질문: “여러분이 고객이라면 어떤 웹서비스가 필요하겠나요?”  
- **Gagné - 주의집중 유도**  
  - 실제 클라이언트 대화 녹음 스니펫 제공  
  - 놀랍게도, 이 서비스는 ‘내가 직접 만든다!’

---

### 2. 🎯 학습 목표 (공유용 출력 강조)  
> **이번 프로젝트에서 여러분은 다음을 할 수 있게 됩니다!**
- 요구사항 정의서를 작성할 수 있다.  
- 워크플로우 및 메뉴구조도를 설계할 수 있다.  
- SQL 기반 테이블을 설계하고 쿼리를 작성할 수 있다.  
- Java(Spring)에서 Controller-Service-DAO 계층을 구현할 수 있다.  
- 나만의 서비스를 팀과 함께 설계하고 피드백할 수 있다.  

---

### 3. 🔁 선수학습 상기 및 실생활 연결  
- **Gagné - 선수학습 회상 유도**  
  - “우리가 HTML/CSS로 만든 웹페이지 기억나나요?”  
  - “이제 그걸 실제 서비스처럼 작동하게 만들어 볼 거예요.”  
- **실생활 연결 예시**  
  - ‘이메일 신청 폼’ → ‘실제 DB에 저장되는 신청 기능으로 발전’

---

### 4. 🔑 핵심 개념 및 키워드 암기 전략  
> **🌟 암기 공식: “요구는 정의되고 흐름은 흐른다”**

| 키워드 | 암기법 |
|--------|--------|
| 요구사항정의서 | 고객의 잃어버린 일기장 (기능 목록) |
| 워크플로우 | 일의 순서도! 계란후라이 순서 기억나죠? |
| 메뉴구조도 | 건물의 층별 안내도 |
| SQL | 데이터 저장소 비밀번호 |
| DAO/Service/Controller | 일꾼(DAO) - 관리자(Service) - 안내자(Controller) |

---

### 5. 🔍 학습 목표별 탐구형 활동  
- **개별 및 팀 탐색형 과제**
  1. 클라이언트 역할과 개발자 역할을 번갈아 가며 인터뷰  
  2. 인터뷰 내용을 바탕으로 요구사항정의서 작성  
  3. 작성한 내용을 기반으로 워크플로우 그리기  
  4. SQL 테이블 설계 후 `CREATE`, `INSERT`, `SELECT` 작성  
  5. Java단 계층 구현 (VO-DAO-Service-Controller)
  6. 간단한 화면 연동 (HTML + Java + DB)

---

### 6. 🤝 실습 활동 및 협력 학습 유도  
- **프로젝트 팀 구성(2~3인)**
- 역할 분담 예시:
  - 기획자(요구사항정의서 및 워크플로우)
  - 개발자(SQL 및 Java)
  - 발표자(결과 요약 및 시연)
- **협업 도구 사용**:
  - Figma, Draw.io(워크플로우)
  - DBDesigner (ERD)
  - Visual Studio Code + Spring Boot
