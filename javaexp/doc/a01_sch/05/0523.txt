## 💡 프로젝트 기반 학습 교안: 데이터베이스 연동 웹 프로젝트 개발

### 1. \[주의 끌기 및 학습 연결성 제공 – *ARCS의 A (Attention)*]

> **✨ “너라면 이 사이트, 어떻게 바꿀래?”**
> 유명 웹사이트(배달앱, 쇼핑몰, 예약 시스템 등)의 현재 구조를 보여주고 “이 사이트를 직접 만든다면? 또는 개선한다면?”이라는 **실제상황 기반의 유쾌한 영상** 또는 \*\*밈(meme)\*\*으로 오프닝
>
> 🔍 *개인화된 질문*:
> “여러분, 내가 만드는 웹사이트에 로그인 안되는 버그가 생겼다면…?”
>
> ✅ 연결: 지금부터 여러분은 실무처럼 직접 사이트를 분석하고, UI 흐름을 설계하며, DB를 연동해 웹서비스를 구축합니다!

---

### 2. \[학습 목표 명확히 제시 – *ARCS의 R (Relevance)*]

> #### 🎯 이번 프로젝트의 목표

1. **요구사항을 분석하고 개선 방향을 스스로 제안**할 수 있다.
2. \*\*HTML, CSS, JS로 사용자 인터페이스(UI)\*\*를 설계하고 구현할 수 있다.
3. **JSP와 DAO로 데이터베이스를 연동**하여 실제 정보를 출력할 수 있다.
4. **웹 페이지의 흐름과 메뉴 구조를 설계**하고 문서화할 수 있다.
5. **완성된 결과물을 실무 시나리오로 설명**할 수 있다.

---

### 3. \[선수 학습 요소 상기 및 실생활 연결]

> 👓 *“우리가 왜 이걸 배우는가?”*
> **HTML은 건축의 벽돌**, **CSS는 페인트와 장식**, **JavaScript는 자동문!**
> **JSP/DAO는 ‘자동문이 잘 열리게 하기 위한 센서’**, 그리고 **DB는 우리 가게의 장부**입니다.

> 🧠 복습 퀴즈 or 활동:
>
> * HTML/CSS/JS 기초 구조 짝 맞추기
> * SQL SELECT\~WHERE 절까지 간단 퀴즈
> * DAO란 무엇인가? 그림으로 표현해보기 (ex: 주전자와 컵의 관계로)

---

### 4. \[핵심 개념/용어를 최고의 암기법으로 제시 – *ARCS의 C (Confidence)*]

| 개념       | 암기 비유       | 설명                    |
| -------- | ----------- | --------------------- |
| DAO      | **배달기사**    | DB와 웹을 연결하는 메신저 역할    |
| Table    | **엑셀 시트**   | 행(레코드)과 열(컬럼)의 구성     |
| SQL      | **주문서**     | 정보를 요청하거나 저장하는 명령서    |
| UI/UX 흐름 | **놀이공원 동선** | 사용자가 웹을 이용하며 이동하는 경로  |
| 요구사항 정의서 | **건축 설계도**  | 무엇을 만들 것인지 미리 약속하는 문서 |

🔁 퀴즈: “DAO는 어떤 배달 기사일까? 배달 사고가 나면 무슨 일이 생길까?”

---

### 5. \[학습 목표별 탐구형 학습 – *ARCS의 R (Relevance)*]

| 단계   | 내용             | 학습 활동                          |
| ---- | -------------- | ------------------------------ |
| 분석   | 기존 사이트 구조 분석   | 장/단점 파악, UI 흐름 그리기, 메뉴 정리      |
| 설계   | 개선 사항 도출       | Figma/손그림으로 UI 개선안 스케치         |
| 구현 1 | HTML/CSS/JS 처리 | 메인 페이지, 로그인/회원가입 UI 구현         |
| 구현 2 | DB연동(DAO, JSP) | 데이터 insert/select, 로그인 기능 구현   |
| 결과   | 전체 흐름 테스트      | insert → DAO → DB → JSP → 출력까지 |

---

### 6. \[실습 활동 및 협력 학습 유도 – *ARCS의 S (Satisfaction)*]

> 👨‍👩‍👧 팀 활동 : 2\~3인 1조, 실제 회의처럼 역할 나누기 (프론트엔드 / 백엔드 / DB 설계자)
> 🎲 협력 미션: 팀별로 “기발하고 실용적인 메뉴”를 포함하여 회원가입+상품등록 기능 구현

> 📌 활동 예시:

* “로그인 후 5초 후에 자동으로 광고가 뜨는 JS 넣어보세요.”
* “상품 등록 시 가격이 0원이면 오류 메시지가 뜨도록 처리하세요.”

### 7. \[단계별 피드백 및 자기평가 기회]

| 피드백 포인트  | 체크리스트 질문                    |
| -------- | --------------------------- |
| 요구분석     | 사용자의 입장에서 생각했는가?            |
| UI 흐름    | 메뉴 구조가 논리적인가?               |
| HTML/CSS | 웹 접근성과 시각적 정리가 잘 되어 있는가?    |
| JS       | 버튼 기능이나 사용자 상호작용이 자연스러운가?   |
| DB 연동    | DAO 구조를 정확히 활용했는가? 오류가 나는가? |

> ✅ **개인 루브릭 제공**: 내가 만족한 점 3가지 / 더 개선하고 싶은 점 2가지 작성

### 8. \[수행평가 및 피드백 제공]

> 📝 **결과물 평가 기준**

1. 요구사항 정의서 명확성 (20점)
2. UI 흐름 설계 (20점)
3. 구현 기능 정상 동작 여부 (30점)
4. DB 연동 및 쿼리 정확성 (20점)
5. 발표 및 실무 적용 가능성 설명력 (10점)

> 🌈 **칭찬 피드백 예시**
> “로그인 구현에 실패했지만, DAO 구조를 끝까지 고민하며 해결한 과정이 멋졌습니다!”
