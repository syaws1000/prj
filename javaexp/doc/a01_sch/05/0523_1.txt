* **활동명**: "개발자 VS 비개발자 퀴즈 대결!"

* **내용**: 학생들에게 다음과 같은 질문을 던집니다.
  “코딩 한 줄 없이 앱을 만들 수 있을까요?”
  → 그러면 Mendix가 어떻게 이걸 가능하게 하는지를 짧은 동영상이나 미니 시연으로 보여줍니다.

* **웃음 포인트**: “자바보다 쉽고, 파이썬보다 빠른! 코딩 알레르기 있는 사람도 할 수 있는 플랫폼!”

## **4. 주요 개념/핵심 키워드를 최고의 암기법에 의해 제시**

> (Gagné 4단계 / ARCS: Confidence)

| 용어           | 설명          | 기억법                           |
| ------------ | ----------- | ----------------------------- |
| Mendix       | 시각적 앱 개발 도구 | “맨\~ 쉽게 하는 디벨롭스!”             |
| Microflow    | 로직 구성 요소    | 마이크로한 흐름으로 논리를 설계해요           |
| Domain Model | 데이터 구조 설계   | “앱의 뼈대를 그리는 청사진!”             |
| Widget       | UI 구성 요소    | “앱의 얼굴!” (Button, Text Box 등) |

## **5. 학습 목표별 탐구형 학습**

> (Gagné 5단계 / ARCS: Confidence, Satisfaction)

| 단계     | 탐구활동                                       |
| ------ | ------------------------------------------ |
| Step 1 | Mendix Studio 로그인 후 앱 생성 (무료버전 사용)         |
| Step 2 | Domain Model 만들기 (예: ToDo Entity, Task 속성) |
| Step 3 | 페이지(Page) 생성 및 위젯 배치                       |
| Step 4 | Microflow로 버튼 클릭 시 동작 구현                   |
| Step 5 | 앱 실행 및 결과 확인                               |

> 🔍 *“이거 진짜 내가 만든 거야?” 하는 감탄 유도!*

## **6. 실습 활동 및 협력 학습 유도**

> (Gagné 6단계 / ARCS: Satisfaction)

### 💡미션: *우리 팀만의 ToDo 앱 만들기!*

* 조별 실습: 업무용, 학습용, 개인용 테마 중 하나를 선택
* 역할 분담: 데이터 설계자 / 화면 디자이너 / 테스트 담당 등
* 발표 시간: 3분간 앱 소개 및 만든 과정 공유

> 😄 *앱 이름을 팀원 이름으로 해도 되고, 엉뚱한 이름(예: “미뤄드림”)도 OK!*
