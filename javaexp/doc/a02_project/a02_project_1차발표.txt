# 프로젝트 발표
1. 발표력
2. 전체 발표자료 질적 향상.

2조 
김현아 
1. 사이트 분석은 아주 좋았음.
2. 업무 프로세스 위주로 파악하는 것을 추천함.[분석]
3. 코드 분석 아주 좋았음.

수고하셨습니다. ^^

이채은
1. 구조와 기능이 효과적으로 표현함.
2. 사이트 장점/개선사항을 정리 잘하셨는데, 구체적으로 어떻게 개선하느냐도 표현하면 더 좋을 듯합니다.
3. 요구사항 id/작성자를 기본적으로 추가 필요함

수고하셨습니다. ^^

 
최윤영 
1. 전체적으로 핵심 항목들은 다 포함되어 알찬 내용이었습니다.
2. 요구사항정의서는 본인이 개발한 내용뿐만 아니라 전체 내용도 추가해주세요
3. 요구사항정의서 비고 부분에 코드는 포함 시킬 필요 없을 것같습니다.
 
수고하셨습니다. ^^


안유리 x	




3조 
이기찬 
1. 요구사항 정의서 요청자 ==> 작성자로 변경  
2. 요구사항정의서는 본인이 개발한 내용뿐만 아니라 전체 내용도 추가해주세요
3. 분석부분을 좀 더 추가적으로 필요합니다(장점/개선사항/프로세스)

수고하셨습니다. ^^


공정배 
1. 요구사항ID는 항목마다 다르게 정의합니다.
2. 코드는 핵심 코드 위주로 정리하고, 설명내용을 포함시켜셔 하시면 될 것같습니다.
3. 분석 내용(장단점/보완사항)에 대한 추가가 있으면 더 좋을 것같습니다.

수고하셨습니다.



5조 발표자 

민지홍
1. 구체적인 웹사이트 구현까지 범위를 넓였습니다.^^
2. 실제적인 구현도 필요하지만 여러가지 사이트에 대한 분석과 경험도 병행할 것을 권유합니다.

수고하셨습니다. ^^
 
	
6조 
신명철
1. 전체적인 항목을 다 갖추어져 있음.
2. 사이트 개선 사항도 추가 필요
3. 요구사항의 기능적 요구사항 정도까지 범위를 기술
4. 설명 내용과 함께 구체적으로 적용된 코드까지 표현 필요

수고하셨습니다. ^^
 
안소영
1. 개선사항에 대한 내용을 표현을 구체적으로 표현됨
2. 코드에 대한 정리를 잘 하셨는데, 이와함께 설명 내용도 표현 필요
	구현화면/핵심코드/설명으로 하나의 페이지에 직관적으로 설명하였으면 함.
3. 요구사항의 기능적 요구사항까지 범위를 잡아도 될 것같음..

수고하셨습니다. ^^
